# iframe 剪贴板兼容复制设计

## 背景与根因

余额兑换工具目前在结果页和历史记录中直接调用 `navigator.clipboard.writeText()`。该接口在顶层 HTTPS 页面中可以正常工作，但跨源 iframe 需要父页面通过 iframe 的 `allow="clipboard-write"` 明确授予权限。

sub2api `v0.1.153` 的自定义页面 iframe 没有声明该权限，因此内嵌页面中的异步 Clipboard API 被浏览器拒绝。工具捕获异常后只显示“复制失败，请手动复制”，没有继续尝试兼容路径。新窗口属于顶层页面，不受该 iframe 权限限制，所以复制正常。

本项目不修改 sub2api。修复必须完全位于独立工具中，并保持现有新窗口行为。

## 目标

- 在 sub2api 跨源 iframe 内点击复制按钮时，兑换码可以写入系统剪贴板。
- 结果页、单条历史记录和“复制全部”使用同一套复制行为。
- 顶层页面继续优先使用现代 Clipboard API。
- 两种复制方式都失败时，保留当前“复制失败，请手动复制”提示。
- 复制过程中不记录、上传或持久化额外的兑换码内容。

## 非目标

- 不修改 sub2api 的 iframe 标签或响应头。
- 不通过 `postMessage` 要求父页面代为复制。
- 不改变兑换流程、身份验证、余额扣减或历史记录格式。
- 不承诺在带有严格 `sandbox` 且不允许脚本或剪贴板操作的第三方 iframe 中复制成功。

## 方案

新增 `src/web/clipboard.ts`，导出统一的 `copyText(text)` 函数。组件不再直接访问 `navigator.clipboard`。

复制顺序如下：

1. 检测页面是否处于 iframe。
2. iframe 中先同步执行兼容复制，确保仍处于按钮点击产生的瞬时用户激活范围内。
3. 顶层页面优先调用 `navigator.clipboard.writeText()`。
4. 顶层页面的现代接口不可用或失败时，再尝试兼容复制。
5. 任一方式成功即返回成功；全部失败才返回失败。

兼容复制使用临时只读 `textarea`：

- 将待复制文本写入 `textarea.value`。
- 使用固定定位将元素移出可视区域，避免页面跳动和布局变化。
- 将元素附加到 `document.body`，选中文本并调用 `document.execCommand('copy')`。
- 在 `finally` 中删除临时元素。
- 尝试恢复复制前获得焦点的元素，并使用 `preventScroll` 避免滚动位置变化。

`document.execCommand('copy')` 仅作为 iframe 和异常环境的兼容路径。顶层页面仍使用现代 Clipboard API。

## 组件行为

`ConversionResult.vue` 和 `HistoryList.vue` 都调用统一函数：

- 返回成功时，继续显示现有“已复制”“已复制兑换码”或“已复制全部记录”。
- 返回失败或抛出异常时，显示“复制失败，请手动复制”。
- 兑换码显示、按钮图标、可访问名称和历史记录内容保持不变。

统一函数负责吸收浏览器剪贴板异常并返回布尔结果，组件只负责展示状态，避免两个组件产生不同的回退行为。

## 安全与兼容边界

- 临时 `textarea` 不使用 `innerHTML`，只设置 `value`，兑换码不会被解释为 HTML。
- 临时元素在每次复制后立即删除，即使复制调用抛出异常也会清理。
- 不将兑换码写入日志、URL、网络请求或新的持久化存储。
- 兼容复制必须从用户点击处理函数同步进入，不能由定时器或后台任务触发。
- 当前 sub2api iframe 未设置 `sandbox`，因此兼容路径适用于当前生产嵌入方式。

## 测试设计

### 单元测试

新增剪贴板工具测试，覆盖：

1. 顶层页面优先使用 `navigator.clipboard.writeText()`。
2. iframe 中不依赖现代接口，兼容复制成功时返回成功。
3. 临时元素在成功和异常路径都会删除，并恢复原焦点。
4. 现代接口和兼容复制都失败时返回失败。

更新组件测试，确认三个复制入口根据统一函数的结果显示成功或失败状态。

### 浏览器回归测试

扩展现有 Playwright iframe 场景。测试父页面继续使用不含 `allow="clipboard-write"` 的跨源 iframe，与 sub2api 当前实现一致。

测试流程：

1. 在 iframe 内完成一笔测试兑换并显示测试兑换码。
2. 点击结果页复制按钮。
3. 从浏览器剪贴板读取内容，确认与兑换码完全一致。
4. 确认页面没有显示复制失败提示，也没有新增浏览器错误。

该测试必须在修复前因 iframe 复制失败而失败，修复后通过。

## 验收标准

- 生产形态的跨源 iframe 中，结果页复制按钮可以复制兑换码。
- 历史记录单条复制和复制全部使用相同兼容能力。
- 新窗口中的现代复制行为不退化。
- 复制后焦点和页面滚动位置不发生明显跳动。
- 两种复制方式都不可用时仍提供明确失败提示。
- 单元测试、iframe Playwright 测试、类型检查和生产构建全部通过。
